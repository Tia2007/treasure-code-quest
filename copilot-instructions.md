# 專案概述
建立一個兒童輸入 3 位數代碼，得到提示或寶藏兌換卡的簡易網頁。

# 目標與使用情境
- 面對的對象是兒童，所以需要簡單易懂的介面與可愛的 UI/UX 設計
- 主要在手機或 iPad 的瀏覽器使用（觸控友善、按鈕要大）
- 最終成果放在 GitHub Pages 上直接展示

# 技術與實作約束（已決策）
- 前端框架：Vite + React
- 不使用 TypeScript（請用 JS / JSX，不要建立 .ts / .tsx）
- 路由：使用 Hash 形式（例如 #/admin、#/play），避免 GitHub Pages 重整造成 404
- 資料保存：使用 localStorage 保存管理員設定、密碼組合、解鎖進度
- 管理員頁面：不做 PIN 或登入保護（直接開放）

# 核心功能

## 管理功能（Admin）
- 管理員可以設定兒童人數（= 幾組密碼）
- 每位兒童有不同的密碼，不能重複
- 密碼為 3 位數字（可包含前導 0，例如 007）
- 管理員的設定（兒童人數、全部解鎖後顯示模式、最終提示文字）採用「先修改、按套用才生效」：
  - 未按「套用」前：只是在 UI 上暫存，不會寫入 localStorage，也不會影響 Play 頁面
  - 按下「套用」後：才寫入 localStorage 並正式生效
- 密碼隨機產生：只有在「兒童人數變更」且按下「套用」時，才會重新產生一整組新的密碼
- 解鎖狀態重置：只有在「兒童人數變更」且按下「套用」時，才會重置解鎖進度（因為密碼已更新）
- 「套用」按鈕互動：
  - 平常 disabled
  - 只有在以下任一情況才會啟用：兒童人數改動、unlockMode 改動、（在提示模式下）最終提示文字改動
- 管理員可以選擇「全部解鎖成功後」顯示內容：
  - A. 寶藏兌換卡
  - B. 最終寶藏提示（提示內容可自行輸入）

## 代碼輸入（Play / Kid）
- 代碼輸入欄位：使用手機/平板的原生數字鍵盤輸入（例如 type="tel" + inputMode="numeric"）
- 輸入內容必須「明碼顯示」（不可用 password 欄位或任何遮罩顯示）
- 避免被瀏覽器/密碼管理器誤判成密碼欄位（例如 autoComplete="off"）
- 不用選擇「小朋友」：輸入任何一組尚未解鎖的 3 位數代碼即可解鎖一格
- 輸入錯誤時顯示固定提示文字（不可設定）

## 解鎖規則
- 只有當「每位兒童都輸入正確的密碼」後才會解鎖寶藏
- 解鎖後顯示：寶藏兌換卡 或 最終寶藏提示（依管理員選擇）

# 追加規格：寶藏兌換卡（Reward Card）
- 不做「上傳兌換卡圖片」功能
- 兌換卡固定顯示：
  - 標題文字：寶藏兌換卡
  - 一張內建圖片：固定使用 public/reward-card-art.png（之後要換圖就直接覆蓋同名檔案，不需改程式碼）
- GitHub Pages 相容性要求：
  - 程式引用圖片時需使用 import.meta.env.BASE_URL + 'reward-card-art.png' 組合路徑（避免 repo 子路徑部署時圖片 404）
- 卡牌外觀（手機/平板）：
  - 呈現為「一張卡牌」的樣子（類似收藏卡的觀感）
  - 直式卡片比例固定為 2:3
  - 圓角、明顯外框與內框，視覺上像一張可收藏的卡

# UI/UX 規範

## 設計原則
- 明亮繽紛配色，適合兒童喜好
- 背景：宇宙主題，增添神秘感
- 圓潤字體，最小 16px
- 行動裝置優先：按鈕大、間距足、可單手操作

## 介面元素與互動
- 版面：上方顯示代碼輸入區，下方顯示寶箱
- 進度顯示：跟寶箱進度放一起，並擺在寶藏上方（例如：進度 2/4 (50%)）
- 寶箱狀態：
  - 初始為灰階寶箱
  - 解鎖過程中的寶箱進度圖固定使用 public/process-chest.png（之後要換圖就直接覆蓋同名檔案，不需改程式碼）
    - 呈現方式：同一張圖「雙圖疊加」
      - 底圖：用 CSS filter 做灰階（grayscale/brightness/contrast）
      - 上圖：保留彩色，使用 clip-path 依進度百分比裁切顯示（像進度條）
    - 進度方向：由左到右
    - 動畫：clip-path 變化需有平滑轉場（例如 transition: clip-path 0.5s ease）
    - GitHub Pages 相容性：引用圖片路徑需使用 import.meta.env.BASE_URL + 'process-chest.png'
  - 每當一位兒童解鎖成功，寶箱的「一部分」變成彩色（代表進度）
  - 當所有兒童都解鎖成功後，寶箱會變成「可點擊翻面的卡片」：
    - 正面：使用一張固定圖片取代原本的寶箱元件圖案，圖片固定使用 public/unlocked-chest.png（之後要換圖就直接覆蓋同名檔案，不需改程式碼）
      - 圖片必須「完整顯示、不裁切」（例如使用 object-fit: contain 的策略）
      - GitHub Pages 相容性：引用圖片路徑需使用 import.meta.env.BASE_URL + 'unlocked-chest.png'
    - 背面：寶藏兌換卡 或 最終提示（依管理員選擇）
- 特效：
  - 每次輸入成功都會有「全螢幕」灑彩紙特效
    - 特效必須覆蓋整個 viewport（full view），不可只在小區塊顯示
    - 噴發位置以「畫面中間」為主（不要左右兩邊噴發）
    - 噴發高度要落在寶箱附近的視覺區域（偏畫面中間略上/略下可微調，但不可太低）
    - 粒子數要夠多、夠明顯（對兒童一眼看得到）
  - 當「全部解鎖成功」的那一次，改用更盛大的「煙火」連發特效（仍然置中、全螢幕）
  - 無障礙：若使用者偏好減少動態（prefers-reduced-motion），特效改為低強度版本（不完全關閉）
    - 同理，寶箱進度的轉場動畫也應縮短/降低強度，而不是完全關閉
  - 全部解鎖成功後，寶箱背景會發光並出現光暈
- 解鎖後顯示：寶藏兌換卡 或 最終寶藏提示（依管理員選擇）
